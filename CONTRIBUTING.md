# Contributing to kanban

Thanks for contributing.

## Ground rules

- Keep changes small and focused
- Prefer incremental, testable slices
- Do not add large framework choices without discussion

## Before opening a PR

```bash
npm run check
npm run build
```

Both commands must pass locally.

## Philosophy

Kanban is in foundation mode. Favor clear primitives and good tooling over early complexity. Build extensibility into the core, then layer product features iteratively.
